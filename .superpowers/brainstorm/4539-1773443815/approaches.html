Project Architecture: 3 Approaches

How to structure this pan-Zen encyclopedia — each approach implies different trade-offs for data, UI, and scale

A

Monolith: Single Next.js App (Recommended)

One Next.js application with everything: data pipeline scripts, API routes, lineage visualizer, encyclopedia pages, i18n — all in one repo.

Pros

  • Simplest to start — no infra overhead
  • Shared types and data models everywhere
  • Easy to deploy (Vercel, one command)
  • You can refactor into services later if needed

Cons

  • Data pipeline mixed with frontend code
  • Could get unwieldy at full scale
  • Build times grow with content volume
B

Dual Repo: Data Engine + Web App

Separate the knowledge base (data pipeline, research scripts, structured JSON/DB) from the web experience (Next.js app consuming the data via API).

Pros

  • Clean separation of concerns
  • Data pipeline can run independently
  • Multiple frontends possible (web, mobile, API)
  • Research phase naturally lives in data repo

Cons

  • More infrastructure to manage
  • Need to define API contracts early
  • Cross-repo type sharing adds friction
C

Monorepo: Shared Workspace

One repo with multiple packages: packages/data, packages/visualizer, packages/web, packages/i18n — managed with Turborepo or pnpm workspaces.

Pros

  • Best of both worlds — separation + shared types
  • Each package can be developed/tested independently
  • Single repo, single CI/CD pipeline
  • Visualizer can be extracted as a library

Cons

  • Monorepo tooling has a learning curve
  • More complex initial setup
  • Can feel over-engineered early on

My Recommendation: Option A (Monolith) to start

Given your "research phase first" approach, starting with a single Next.js app keeps things simple. The data pipeline lives as scripts in the same repo. When the project grows, you can extract packages naturally. Premature separation adds friction without benefit at this stage.

Phase 1: Data model + research scripts + seed data
Phase 2: Lineage visualizer prototype (animated flow)
Phase 3: Full encyclopedia pages + i18n
Phase 4: Timeline experience + polish
