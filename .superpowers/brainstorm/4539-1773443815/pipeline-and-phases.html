Research Pipeline & Project Phases

Data first, then build — the knowledge base is the foundation

Data Collection Pipeline

Step 1 — Source

Extract Lineage Data from Existing Charts

Parse the PDF lineage charts you've collected, plus scrape the reference websites (Tibetan Buddhist Encyclopedia, Terebess, Cosmos Chan, Mountain Moon). Extract: master names (multilingual), dates, teacher→student relationships, school affiliations.

Output: raw/lineage-charts/*.json

Step 2 — Reconcile

Merge & Deduplicate Masters

Same master appears with different romanizations across sources (Dōgen / Dogen / Tao-yuan / 道元). Build a reconciliation script that matches by: dates, Chinese characters, known aliases. Human review for ambiguous cases.

Output: data/masters.json — canonical master list with all name variants

Step 3 — Enrich

AI-Assisted Content Generation

For each master, use Claude to generate biographical content from scholarly sources:

  • Biography (life, training, enlightenment, teaching career)
  • Key teachings and their significance
  • Historical context (dynasty, political events, Buddhist climate)
  • Temple associations with locations
  • Connections to broader Buddhist traditions

Output: data/profiles/*.json — rich profiles per master

Step 4 — Translate

Multi-Language Content

Generate translations for all content in target languages (EN, JA, ZH, KO, VI). Use AI translation with terminology consistency checks — Buddhist terms must use standard translations per language.

Output: data/i18n/[locale]/*.json

Step 5 — Curate

Human Review & Verification

You review AI-generated content for accuracy, add personal insights, correct errors, flag uncertain claims. A curation UI or simple markdown workflow where you can approve, edit, or flag each profile.

Output: data/reviewed/*.json — verified content ready for production

Project Phases

Phase 1 START HERE

Data Foundation

Set up the Next.js project, define TypeScript data models, build the extraction and reconciliation pipeline. Populate the database with lineage data from all source charts. Establish the canonical master list with multilingual names and teacher→student relationships.

Deliverables: Project scaffold, data models, extraction scripts, ~500+ masters with lineage links, all three major lineages (Soto, Rinzai, Sanbo) mapped
Phase 2

Lineage Visualizer MVP

Build the core interactive lineage explorer with sumi-e aesthetics. D3.js for layout + Canvas/SVG rendering. Implement zoom, pan, time scrubber, and basic node interactions. Master sidebar with essential bio on click.

Deliverables: Interactive lineage tree, animated flow rendering, time navigation, school color coding, sidebar details panel
Phase 3

Encyclopedia Content

AI-assisted profile generation for all masters. Build rich profile pages, school pages, teachings browser. Implement i18n infrastructure and generate multi-language content. Add temple map with Mapbox/Leaflet.

Deliverables: Full master profiles, school pages, teaching/koan browser, temple map, multi-language support
Phase 4

Timeline & Polish

Build the immersive scrollytelling timeline. Add historical events, cultural context, connections to broader Buddhism and Taoism. Search system, glossary, SEO optimization. Performance tuning for the visualizer with large datasets.

Deliverables: Timeline experience, global search, glossary, performance optimization, deployment
